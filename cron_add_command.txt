/cron add '0 9 * * *' "## 任务：每天自动监控 arXiv 论文并推送飞书

### 第一步：执行 Python 脚本
cd 到项目根目录后，用 python3 运行：
python3 monitor.py

### 第二步：判断是否有新论文
运行完毕后检查脚本输出：如果包含 'No new papers'，说明今日无新论文，直接回复：✅ 今日（YYYY-MM-DD）未发现新的论文。
如果有 'LLM_SUMMARIZATION_REQUIRED' 部分，说明有新论文，进入第三步。

### 第三步：读取 JSON 并完成 LLM 信息补全
读取 new_papers.json，该文件包含 new_count 和 new_papers 数组。

对每篇论文必须完成以下两步：

1）提取作者单位 affiliations
- 使用 pdfplumber 或 PyMuPDF 读取 PDF（路径在 pdf_local_path 字段）前 2 页文本，找到作者名对应的机构信息。
- 提取与清洗规则：
  - 优先使用上标脚注映射（如 ^1、^2 对应单位）
  - 如无清晰脚注，再匹配常见机构名（MIT、Stanford、Google、Microsoft、Tsinghua、Peking、Intel、NVIDIA、IBM 等）
  - 对连写英文做分词还原，例如 DepartmentofPoliticalSciences -> Department of Political Sciences
  - 删除明显噪音：URL、正文叙述句、公式片段、参考文献片段、无关硬件说明等
  - 若提取内容包含连续 18 个以上英文字母且无空格，视为未清洗完成，必须再次清洗
  - 输出格式：多个单位使用 ; 分隔，不要换行，不要保留脚注编号
  - 如果确实无法确定单位，填 未找到单位信息

2）生成中文总结 summary_cn
- 基于每篇论文 abstract 生成一段中文总结，必须满足：
  - 长度 90-150 个中文字符
  - 必须覆盖方法核心、主要贡献、关键结果或效果
  - 禁止模板化空话，禁止仅写"提出新型量化方法"等泛化句
  - 只输出一段自然中文，不使用分点
- 质量自检：若 summary_cn 少于 90 字，或与其他论文摘要高度雷同，重写；若 summary_cn 包含明显空泛模板短句，重写。

### 第四步：更新 Excel
用 openpyxl 将 affiliations 和 summary_cn 回填到 papers_record.xlsx 的对应行（按 arxiv_id 匹配）。

### 第五步：构建飞书 Markdown 消息
输出格式如下：
📚 **论文日报** | YYYY-MM-DD
共发现 **N** 篇新论文
每篇包含：论文标题、arXiv ID、日期、作者、单位、PDF 下载链接、90-150 字中文摘要

最后附：PDF 已下载至本地 papers/，记录已更新至 papers_record.xlsx。

重要：最终回复只输出飞书消息文本，不要输出其他说明。"
